### Libraire vs Frameworks

**Librairie**
+ Une collection de code réutilisable
+ Réduit le besoin d'écrire des choses complexes et répétitives en partant de zéro
+ Offre une grande liberté d’utilisation (possible de contrôler comment et quand elle est utilisée dans un projet)
+ Peu de contraintes sur la structure du projet

**Framework (cadriciel)** 
+ Architecture prédéfinie : Il faut suivre un "pattern" de développement (*une façon de développer*) spécifique au *framework*.
+ Il faut travailler dans les limites établies par le *framework*.
+ "Bonnes" et "mauvaises" façons d'utiliser un *framework*.
+ Un framework peut utiliser des librairies
